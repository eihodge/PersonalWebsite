Messaging Application

Introduction

This messaging application is designed to facilitate real-time communication between users. This project aims to demonstrate my skills in both frontend and backend development, as well as database design and integration. I built this project using React, JavaScript, HTML, and CSS for the frontend, and for the backend I used Flask, Flask_SQLAlchemy, and Flask_CORS. The database is built using SQLite, and the application is deployed on a Flask server.

High Level Design

This high-level design diagram shows the architecture of my messaging application, visualizing how the frontend, backend and database interact.

The Frontend, which is built with React, handles the user interface and sends requests to the Backend Server using RESTful API calls.

The Backend Server, which is implemented with Flask, processes requests and interacts with the SQLite Database to manage users, conversations and messages.

Database Schema

This Entity Relationship Diagram visualizes the design of my database schema. There are three tables: User, Conversation, and Message. Each Conversation involves two Users, linked through the two foreign keys 'user1_id' and 'user2_id'. Each Message is linked to a single conversation with the 'conversation_id' foreign key, as well as to two users through the 'sender_id' and 'receiver_id' foreign keys. Each Message also has a 'content' string to store the text of the message, as well as a DateTime 'timestamp' of when the message was sent. 'timestamp' is used for sorting and displaying to the user.

API Endpoints

User

Endpoint HTTP Method Parameters
/users GET None
/create_user POST username, password
/delete_user/<user_id> DELETE user_id (URL Parameter)
/user/<username> GET username (URL Parameter)

Conversation

Endpoint HTTP Method Parameters
/create_conversation POST user1_id, user2_id
/conversations/<user_id> GET user_id (URL Parameter)

Message

Endpoint HTTP Method Parameters
/messages/<conversation_id> GET conversation_id (URL Parameter)
/send_message POST sender_id, receiver_id, content, conversation_id, timestamp
